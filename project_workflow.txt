Setting up AWS Account --
1. Log into AWS management console
Can be done as a root user
2. Select region -- US East (N. Virginia) us-east-1 OR US West (Oregon) us-west-2 to access newest models of Amazon Bedrock etc.
3. Set up billing alram -- budgets --> create budget --> simplified and zero spend budget --> mention email -->save
4. Amazon Bedrock 
amazon bedrock --> model catalog --> anthropic model sonnet 4 --> open in playground --> fill the form (only first time) --> chat/text playground --> type Hello and see if the model is working
5. Creating our IAM user
IAM --> users --> create users --> "select a name" then do attach policies directly --> administrator access --> select next and click on 'create user'
Click on this user --> ''security credentials'' tab --> access keys --> 'create access key' -->select 'local code' --> click next --> copy the 'access key id' and 'secret access key' immediately


Connecting the laptop to AWS -- 
1. Create one .env file 
2. Put up this, 
    AWS_ACCESS_KEY_ID=your_access_key_here
    AWS_SECRET_ACCESS_KEY=your_secret_key_here
    AWS_DEFAULT_REGION=us-east-1  # Or us-west-2, depending on where you requested Bedrock access


Setting up LangSmith for Observability and triggering the bedrock model from local
1. Create a python venv -- python -m venv genai_venv
2. Activate the venv and select this venv as interpreter in VSCode
3. pip install -r requirements.txt
4.  Set up LangSmith --
    Signup with github on smith.langchain.com
    Settings --> API keys --> Create API key --> copy the key
    Add this key in .env file
5. Write test_bedrock.py to check if my laptop can successfully authenticate with AWS and trigger Bedrock model
6. Check the langsmith dashboard to see the visual trace of API calls that we made ny running test_bedrock.py

Creating a local database for structured data and S3 bucket where unstructured PDF data would be uploaded
1. Create a local db using setup_db.py and verify it in the local folder
2. Set up S3 bucket 
    S3 --> create bucket --> give a globally unique name --> leave this checked, 'block all public access' -->create bucket. 
    Keep the unique bucket name handy to mention in the script where S3 bucket will be used.
3. Have a sample_policy.pdf in the project folder
4. Write a script 'document_utils.py' to extract text from the pdf using PyMuPDF and then upload the pdf to s3 bucket. 
    Test it and manually check S3 bucket if data is there.

Creating rag tool
1. Write rag_pipeline.py which loads a pdf, chunks it, creates embeddings and puts them in local chrome db.
    This rag tool will be used by supervisor agent to route between this or structured data stored in SQLite db.
    Test this rag tool standalone.

Creating the supervisor agent 
1. Write agent.py which based upon query goes to either SQLite db tool or vector db tool or both of them and synthesizes the final answer and returns.

Creating FastAPI and Observability
1. Write main.py to create endpoints/APIs for uploading the pdf to S3 and also asking queries to custom agent.
2. Since FastAPI requires a web server Uvicorn to run so run this code using this command, "uvicorn main:app --reload"
3. Access the swagger UI at http://127.0.0.1:8000/docs or http://localhost:8000/docs and ask any query in /ask-custom-agent endpoint.
4. Check the traces at langsmith dashboard
5. Can close the server by ctrl+c
